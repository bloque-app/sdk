# Principles — Personality, Voice-in-Design, Rules (Light)

The soul of the system. These are **identical** to the dark-mode principles — light is not a different brand, it's the same brand on a different canvas.

## Brand personality (same as dark)

> **Premium, engineered, trustworthy.** Bloque is a precision instrument for moving money. Not playful, not corporate, not crypto-bro. The feeling of a well-machined tool — confidence radiating from competence.

Design goal: **excitement within 5 seconds** — *"holy shit, I need this."* Not hype. Earned excitement from seeing something genuinely well-built that solves a real problem.

Audience: **mixed** — developers (indie hackers, startup engineers, AI builders) AND non-technical decision-makers (CFOs, finance leads, ops). Every surface must speak to both without dumbing down for either.

## Five design principles (same as dark)

1. **Density signals competence.** Don't hide information behind "simplicity." Show the product's depth confidently. A light canvas can feel airy — resist the temptation to use all that white space as filler. Bloque's light surfaces are still dense.

2. **The form IS the product.** Qualification flows, pricing configurators, SDK demos — the interaction itself should feel like using Bloque.

3. **Warm purple-tinted near-white.** Every surface carries the purple undertone that ties it to `#0d0c17`. Never pure white, never neutral gray, never cream. The canvas is `#faf9fc` — recognizably the same brand as dark, just inverted.

4. **Surgical accent.** Deep violet `#7c3aed` appears in exactly **2–3 places per viewport**. If you see a fourth, remove one. Soft violet `#a78bfa` can appear more freely as decorative tint — but never as text or CTA.

5. **Mixed audience, single register.** Same rule, same voice, same copy standards.

## Light-mode specific: stay recognizable

A light Bloque surface and a dark Bloque surface placed side-by-side must read as the same brand. The tells are:

- Same typography (compressed display, clean sans body, mono labels)
- Same scaffold (eyebrow + headline + content)
- Same copy voice (one italic accent word per headline)
- Same density
- Purple undertone on every surface

If the light version reads like a generic light SaaS while the dark reads like Bloque, the light version is wrong. Match the soul, not just the colors.

## The three anti-references (same as dark)

Bloque is defined as much by what it isn't as what it is. Never let a surface drift toward any of these:

### ❌ NOT Web3 / crypto
- No neon pink/green gradients
- No "to the moon" energy
- No glassmorphism for its own sake
- No 3D chrome / glossy effects
- No degen typography

### ❌ NOT generic SaaS template
- No hero → 3-column features → pricing grid cliché
- No stock illustrations of smiling cartoon people
- No "empower your business" copy
- No pastel gradient CTA buttons
- No unearned testimonial carousels
- **Light-specific warning**: this is the trap the light theme falls into most easily. A light page with generic copy and a stock hero image is indistinguishable from 10,000 SaaS pages. Bloque's light theme must work harder than its dark theme to resist this.

### ❌ NOT traditional banking
- No navy blue anywhere
- No stock photos of handshakes or skyscrapers
- No "trust" iconography (shields, locks as decoration)
- No stuffy serif headlines set large
- No beige / cream / conservative palette — **`#faf9fc` is not cream**. If your canvas drifts toward warm yellow, you're in banking territory. Pull it back toward the cool purple undertone.

## The voice-in-design principle (same as dark)

The copy voice IS part of the design. A Bloque page without a compressed-tracking headline that uses one italic accent word will feel wrong even if every token is correct. On light, the italic word takes **`#7c3aed`**.

- *Finance is the **last** frontier.*
- *The builder of your financial stack should be **you**.*
- *Design **your own**.*

## What "precision instrument" looks like in light

- Tight alignment. Elements line up on clear grids.
- Consistent tracking. Same headline size → same negative letter-spacing.
- Calm motion. 150–250ms, ease-out, no bounce.
- No stock iconography. Line-based icons, 1.5–1.75px stroke, muted foreground color, never the accent.
- No filler. If a section doesn't do work for the page, cut it.

**Light-specific**: the temptation on light surfaces is to rely on shadows and heavy borders to define structure. Resist. Structure comes from tight alignment, surface-tone steps, and the gradient hairline divider — not from box-shadow.

## When you're stuck

Ask: *would Linear ship this?* (Linear has both light and dark — in light they maintain the same precision feel.) If the answer is no, the design is drifting.

Then ask: *if I placed this next to the dark version of the same page, would they clearly read as the same brand?* If not, the light version has lost the thread. Usually the fix is:

1. Cool the canvas (more purple undertone, less warmth)
2. Deepen the accent moments (use `#7c3aed` for text, not `#a78bfa`)
3. Tighten the tracking (light headlines tend to read looser than dark)
4. Restore the density (light space reads as more empty than dark space)
