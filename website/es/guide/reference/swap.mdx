# Swap

<SwapFlowDiagram />

El mĂłdulo swap es la frontera entre los rieles de pago externos y las cuentas Bloque. Ăsalo para consultar tasas de cambio y crear Ăłrdenes de cash-in o cash-out.

## Medios Soportados

| Medio | DirecciĂłn | DescripciĂłn |
| --- | --- | --- |
| `pse` | `fromMediums` (solo cash-in) | Transferencia bancaria en tiempo real colombiana (23+ bancos) |
| `kusama` | `fromMediums` o `toMediums` | Red blockchain Kusama |
| `breb` | `fromMediums` o `toMediums` | Llave de pago instantĂĄneo BRE-B |
| `rtp` | `toMediums` (solo cash-out) | Pago instantĂĄneo a banco US vĂ­a RTP |
| `external-us-bank` | `fromMediums` (solo cash-in) | On-ramp ACH desde banco US |
| `bankTransfer` | `toMediums` (solo cash-out) | Cualquier banco colombiano (29 bancos) |
| `bancolombia` | `toMediums` (solo cash-out) | Cuenta bancaria Bancolombia |

Usa `fromMediums` para especificar cĂłmo entran los fondos al swap, y `toMediums` para indicar dĂłnde llegan.

## DescripciĂłn General

El mĂłdulo swap permite consultar tasas de cambio y realizar intercambios de activos entre diferentes medios y monedas soportadas.

### Pagos parciales (subpago)

Si el pagador envĂ­a **menos** que `order.fromAmount`, el grafo **sigue pausado** en la **misma** `keyValue` y `callbackToken`. Varias transferencias BRE-B a esa llave se **suman**. Tras cada pago, `how` se actualiza:

| Campo | Primera pausa | Tras pago parcial |
| --- | --- | --- |
| `depositStatus` | `'awaiting'` | `'partial'` |
| `receivedAmount` | `'0'` | COP escalado recibido hasta ahora |
| `remainingAmount` | igual a `expectedAmount` | COP escalado aĂșn requerido |
| `keyValue` | llave de un solo uso | **sin cambios** â envĂ­a complementos a la misma llave |

```typescript

  console.log(`Recibido ${how.receivedAmount}, faltan ${how.remainingAmount}`);
  console.log('Complementar en la misma llave:', how.keyValue);
}
```

**Seguimiento:** el servidor acumula `received_total_scaled` en el estado de la instrucciĂłn. `remainingAmount` siempre se deriva (`expectedAmount â receivedAmount`).

**Polling:** suscrĂ­bete a webhooks `swap.order.*` o consulta `user.swap.listOrders({ graphId: result.order.graphId })`. La respuesta inicial de `createDeposit` no se actualiza cuando llega un pago parcial posterior.

### Sobrepago

Si el pagador envĂ­a **mĂĄs** de lo requerido, el swap entrega el `order.toAmount` DUSD **cotizado**. El exceso en COP se marca para reembolso manual; no se convierte en DUSD extra.
