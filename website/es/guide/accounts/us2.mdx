# Cuentas US2 (Kira)

Crea y gestiona cuentas bancarias de Estados Unidos usando el nuevo flujo `accounts.us2`.

## Descripción general

`accounts.us2` es independiente del cliente `accounts.us` respaldado por Bridge. Ambos medios existen al mismo tiempo durante la migración:

- **`accounts.us`** mantiene el flujo de Bridge con `getTosLink()` y `signedAgreementId`
- **`accounts.us2`** usa onboarding guiado por cumplimiento y no requiere link de TOS

Este primer alcance del SDK soporta solo **`type: 'individual'`**. El onboarding empresarial queda diferido intencionalmente.

## Diferencias clave frente a `accounts.us`

- No hay paso de `getTosLink()`
- No existe `signedAgreementId`
- `proofOfAddress` es obligatorio
- Las instrucciones bancarias pueden aparecer después, cuando termine la provisión asíncrona
- `virtualAccountId` y `sourceDepositInstructions` no están garantizados inmediatamente después de `create()`

## Crear una cuenta

```typescript title="crear-cuenta-us2.ts"
import { SDK } from '@bloque/sdk';

const bloque = new SDK({
  origin: 'tu-origen',
  auth: {
    type: 'apiKey',
    apiKey: process.env.BLOQUE_API_KEY!,
  },
  mode: 'production',
});

const account = await bloque.accounts.us2.create({
  type: 'individual',
  email: 'roberto.gonzalez@example.com',
  phone: '+12125551234',
  taxId: '123-45-6789',
  address: {
    street: '456 Wall Street',
    city: 'New York',
    state: 'NY',
    postalCode: '10005',
    country: 'US',
  },
  proofOfAddress: {
    document_type: 'utility_bill',
    document_front: 'base64_encoded_image_here',
  },
  webhookUrl: 'https://api.example.com/webhooks/account-events',
  metadata: {
    source: 'mobile_app',
    onboarding_version: 'us2-v1',
  },
});

console.log(account.urn);
console.log(account.status);
console.log(account.virtualAccountId);
console.log(account.sourceDepositInstructions);
```

## Parámetros de creación

```typescript title="tipos.ts"
interface CreateUs2AccountParams {
  holderUrn?: string;
  type: 'individual';
  email: string;
  phone?: string;
  taxId?: string;
  address?: {
    street: string;
    city: string;
    state: string;
    postalCode: string;
    country: string;
  };
  proofOfAddress: Record<string, unknown>;
  name?: string;
  webhookUrl?: string;
  ledgerId?: string;
  metadata?: Record<string, unknown>;
}
```

:::warning Solo individuales
La superficie actual del SDK solo acepta `type: 'individual'` en `accounts.us2.create()`. El soporte para empresas todavía no se expone intencionalmente.
:::

## Forma de la respuesta

```typescript title="tipos.ts"
interface Us2Account {
  urn: string;
  id: string;
  medium: 'us2-account';
  userId: string;
  virtualAccountId?: string;
  type: string;
  currency: string;
  sourceDepositInstructions?: {
    currency?: string;
    bankName?: string;
    bankAddress?: string;
    bankRoutingNumber?: string;
    bankAccountNumber?: string;
    bankBeneficiaryName?: string;
    bankBeneficiaryAddress?: string;
    clabe?: string;
  };
  status:
    | 'creation_in_progress'
    | 'active'
    | 'disabled'
    | 'frozen'
    | 'deleted'
    | 'creation_failed';
  ownerUrn: string;
  ledgerId: string;
  webhookUrl: string | null;
  metadata?: Record<string, unknown>;
  createdAt: string;
  updatedAt: string;
  balance?: Record<string, { current: string; pending: string; in: string; out: string }>;
}
```

## Provisión asíncrona

Las cuentas US2 son asíncronas por diseño. Un `create()` exitoso significa que el registro de la cuenta existe, no que las instrucciones bancarias ya estén disponibles.

Ciclo de vida típico:

- `creation_in_progress`
- `active`
- `creation_failed`
- `disabled`

Cuando el flujo asíncrono del proveedor termina, llamadas posteriores a `get()` o `list()` pueden empezar a devolver:

- `virtualAccountId`
- `sourceDepositInstructions.bankRoutingNumber`
- `sourceDepositInstructions.bankAccountNumber`
- otros campos de instrucciones de depósito

## Listar cuentas

```typescript title="listar-cuentas-us2.ts"
const result = await bloque.accounts.us2.list();

for (const account of result.accounts) {
  console.log(account.urn, account.status, account.medium);
  console.log(account.sourceDepositInstructions?.bankRoutingNumber);
}
```

## Actualizar metadata

```typescript title="actualizar-metadata-us2.ts"
const updated = await bloque.accounts.us2.updateMetadata({
  urn: 'did:bloque:mediums:us2-account:account:123e4567',
  metadata: {
    updatedBy: 'admin',
    updateReason: 'customer_request',
  },
});

console.log(updated.metadata);
```

## Helpers de estado

```typescript title="gestionar-estado-us2.ts"
const accountUrn = 'did:bloque:mediums:us2-account:account:123e4567';

await bloque.accounts.us2.activate(accountUrn);
await bloque.accounts.us2.freeze(accountUrn);
await bloque.accounts.us2.disable(accountUrn);
```

:::note Semántica de estado
El SDK expone los mismos helpers genéricos de estado que otros clientes de cuentas, pero el comportamiento subyacente del proveedor para congelar o reactivar no es una operación nativa dedicada de Kira. Trátalos como actualizaciones genéricas de estado de cuenta.
:::

## Cuándo usar `us` y `us2`

- Usa **`accounts.us`** cuando necesites el flujo actual respaldado por Bridge
- Usa **`accounts.us2`** para el nuevo onboarding respaldado por Kira y la provisión bancaria asíncrona
- No asumas que ambos flujos devuelven los mismos campos
